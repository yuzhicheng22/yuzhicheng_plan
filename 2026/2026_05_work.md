## 2026_05_06 Wed.

### + sd_unity项目帮会捐赠功能开发遇到的坑记录(只包含框架相关) #work #bug #sd/unity

无限滑动列表的实现并不是很好，开发的过程中要注意

1. 套用的模板预制体需要在父节点设置`temp`标签，否则在生成 UI 的时候会生成不必要的代码；
2. `UILoopListViewContainer.Init(args)`：参数只能为对象，并且使用滑动列表时，初始化回调是`RefreshCallback()`，默认不会自动生成，因为只有滑动列表需要使用，记得重写；

另外创建模板变量的代码
```Csharp

UIUtility.CreatUITemp<XXXTemp>(baseUI, gameObject);

UIUtility.CreatUITemp<ProgressCommonItemTemp>(this, progressTemp.gameObject);

```

GM 中物品生产的功能缺失，只能使用命令 `@生产 itemID 数量`（2026_05_18更新：已解决）

生成配置表数据和协议相关数据在 `Unity -> Tools -> Data Build`中，生成相关配置后处理报错再提交，需要在群里同步通知状态

模版类中需要交互的 `UIControl`在写路径的时候记得检查命名大小写等，后续使用 AI 工具应该可以避免错误；
